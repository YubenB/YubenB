## ⚙️ What I Build

I’m a full-stack engineer with a backend-first mindset.

Not the “just make the button work” type of full-stack.

More like:

```txt
API contracts        → clean
database schema      → intentional
background jobs      → traceable
message queues       → reliable
deployment           → reproducible
production incident  → let's debug it
```

My work usually sits around systems that need to talk to other systems:

* REST APIs and service integrations
* Multi-provider webhook processing
* Event-driven pipelines with RabbitMQ / Kafka
* PostgreSQL schema design and optimization
* Agentic AI workflow automation
* RAG / local LLM integration
* Dashboard and internal tools
* Linux server, Docker, Kubernetes, and deployment workflows

## 🧩 Engineering Identity

I like working close to the ugly part of software.

The part where:

* business logic is not simple
* third-party APIs behave differently
* data shape changes halfway through the project
* queues fail silently
* deployment breaks because of one cursed environment variable
* clients ask for “small changes” that are actually architectural decisions

That’s where I learn the most.
